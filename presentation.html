Zero-Knowledge Auth

We do not store passwords. Authentication is a cryptographic challenge.

1. User inputs Password.
2. App derives KEK (Key Encryption Key) via PBKDF2.
3. App attempts to unwrap stored DEK (Data Encryption Key).
4. Success = DB Unlocked.
5. Failure = Invalid Password.

Data Privacy

Custom Dexie.js Middleware automatically encrypts sensitive fields before storage.

  • Products: Cost Prices & Margins hidden from database inspectors.
  • Customers: PII (Phone, Email, Address) encrypted at rest.
  • Sales: Profit calculations protected.
